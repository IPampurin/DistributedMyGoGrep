## Файлы для DistributedMyGoGrep - распределённой CLI-утилиты grep с кворумом и concurrency  

### 📋 Описание проекта  

Этот проект реализует распределённую версию классической CLI-утилиты grep.  
Основная цель - продемонстрировать работу с параллелизмом (горутины, каналы),  
сетевым взаимодействием (HTTP) и отказоустойчивостью через механизм кворума.  

#### Утилита может работать в трёх режимах:  
- Локальный режим - обычный grep, обрабатывающий файл или stdin.  
- Режим воркера - запуск одного или нескольких HTTP-серверов, которые принимают задачи от мастера.  
- Режим мастера - координатор, который читает входные данные, разбивает их на части (шарды),  
распределяет задачи между воркерами, собирает результаты и выводит итог, используя кворум (N/2+1 успешных ответов).  

#### Особенности:
- Репликация - каждый шард отправляется на несколько воркеров (фактор репликации задаётся через  
переменную окружения REPLICATION_FACTOR_MYGOGREP).  
- Кворум - шард считается обработанным, когда успешные ответы получены от replicationFactor/2 + 1 воркеров.  
- Перераспределение - если первоначальные реплики не дали кворума, задача автоматически пересылается  
на другие, ещё не использованные воркеры.  
- Автоматический запуск воркеров - мастер при необходимости запускает недостающие воркеры как дочерние процессы.  

### 🖥️ Возможности  

#### Поддерживаемые флаги (аналоги grep):  

  -A N - вывести N строк после совпадения  
  -B N - вывести N строк до совпадения  
  -C N - вывести N строк контекста (до и после)  
  -c - вывести только количество совпадений  
  -i - игнорировать регистр  
  -v - инвертировать фильтр (выводить строки, не содержащие шаблон)  
  -F - трактовать шаблон как фиксированную строку, а не регулярное выражение  
  -n - выводить номер строки перед каждой найденной строкой  

#### Режимы работы:  

1. Локальный режим  

```bash
    ./mygogrep -i banana test1.txt
```

2. Режим воркера (ноды) запускает HTTP-сервер(ы) на указанных адресах.  
Воркеры слушают POST-запросы на /task и возвращают результат.  

```bash
    ./mygogrep --addr localhost:9090,localhost:9091
```

3. Режим мастера читает файл (или stdin), разбивает его на шарды (по умолчанию len(workers)*2),  
отправляет задачи, дожидается кворума и выводит результат.  

```bash
    ./mygogrep --cluster localhost:9090,localhost:9091,localhost:9092 -i banana test1.txt
```

#### Фоновые возможности:  

- Мастер автоматически запускает недостающие воркеры (если они не отвечают по TCP).  
- При сбое воркера мастер может перераспределить задачу на другие узлы.

### 🚀 Быстрый старт  

**Требования:**  

  - Go 1.24+
  - Свободные порты для воркеров (например, 9090, 9091, 9092)

**Сборка и запуск:**  

Клонируйте репозиторий:  

    git clone https://github.com/IPampurin/DistributedMyGoGrep.git
    cd DistributedMyGoGrep

Соберите бинарный файл или воспользуйтесь готовым в корне проекта:

    go build -o mygogrep ./cmd/main.go

Запустите кластер из трёх воркеров:  

    ./mygogrep --addr localhost:9090,localhost:9091,localhost:9092

Запустите мастер для обработки файла:  

    ./mygogrep --cluster localhost:9090,localhost:9091,localhost:9092 -i apple ./tests/test1.txt

### ⚙️ Конфигурация  

Доступные настройки задаются через файл .env в корне проекта.

    REPLICATION_FACTOR_MYGOGREP=1   # количество реплик при работе в многосерверном режиме по умолчанию
    COUNT_SCHARD_MYGOGREP=3         # количество шардов, на которые делим данные, по умолчанию

### 📊 Примеры использования  

1. Поиск строки "apple" с игнорированием регистра и выводом номеров строк:  

```bash
    ./mygogrep -i -n apple ./tests/test1.txt
```

2. Распределённый поиск строки "banana" с контекстом 1 строка до и после:  

```bash
    ./mygogrep --cluster localhost:9090,localhost:9091,localhost:9092 -C 1 banana ./tests/test1.txt
```

3. Подсчёт количества строк, содержащих "apple", с использованием трёх воркеров:  

```bash
./mygogrep --cluster localhost:9090,localhost:9091,localhost:9092 -c apple ./tests/test1.txt
```

### 🧪 Сравнение с оригинальным grep  

В тестовом файле tests/test1.txt (2000 строк, 3 совпадения с "apple") запускался  
оригинальный grep и распределённая версия на трёх локальных воркерах.  

Результаты:  

    Утилита                    Время (сек)       Память (МБ)  
    grep (оригинал)               0.008            1.2  
    mygogrep (распределённый)     0.025            5.8  

  - **Примечание**: распределённая версия медленнее из-за накладных расходов на HTTP-вызовы и сериализацию,  
  однако для больших файлов (гигабайты) распараллеливание на нескольких машинах может дать выигрыш.  

### 📌 Особенности реализации  

Горутины и каналы - отправка задач на воркеры и сбор результатов происходит асинхронно.  
Кворум - используется replicationFactor/2+1, что гарантирует устойчивость к сбоям отдельных воркеров.  
Перераспределение - при недостижении кворума задача отправляется на другие воркеры (круговая схема).  
Автоматический запуск воркеров - мастер запускает недостающие узлы как дочерние процессы, что упрощает развёртывание на одной машине.
